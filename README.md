# 기억 인터뷰 MVP

집에서 테스트할 수 있는 Next.js 기반 마음 기록 MVP입니다. 흐름은 `사전 정보 입력 -> 기억의 방 입장 -> 음성 안내 인터뷰 -> 답변 기록 -> 편지 톤/길이 선택 -> 가족에게 남기는 편지 생성`입니다.

## 주요 기능

- 입장 전 정보 입력: 고객 이름, 대상자 이름, 관계, 현재 상태, 기록 목적, 먼저 떠오르는 기억, 꼭 전하고 싶은 말
- 입장 후 3초 동안 “기억의 방에 입장했습니다” 화면 표시
- 고객명과 대상자명을 포함한 차분한 환영 멘트 생성
- 웰컴 화면과 인터뷰 화면에 `기억 진행자` 아바타 카드 표시
- 음성 재생 중 아바타 주변에 은은한 pulse 애니메이션 표시
- Web Speech API `speechSynthesis`로 한국어 음성 우선 재생
- 질문 표시 시 자동 음성 재생, `다시 듣기`, `음성 끄기/켜기` 제공
- 인터뷰 답변을 텍스트 입력 또는 `마이크로 말하기`로 입력
- Web Speech Recognition API를 사용해 한국어 음성인식 `ko-KR` 적용
- 인터뷰 중 `영상 기록 시작`으로 카메라와 마이크를 함께 녹화
- 녹화된 `webm` 영상을 브라우저에서 재생하고 다운로드
- 한 번에 하나의 질문만 표시
- 답변을 평가하거나 과하게 해석하지 않고, 짧게 받아준 뒤 다음 질문 생성
- 총 7개 질문 진행
- 7번째 답변 후 편지 톤과 길이 선택
- 최종 편지 화면에서 `더 부드럽게`, `더 담담하게`, `더 감성적으로`, `짧게 줄이기`, `가족에게 말하듯 수정` 지원
- `/result/[uuid]` 결과 페이지 URL 생성
- 결과 페이지에서 `링크 복사`, `PDF 저장`, `카카오톡 공유` 제공
- PDF 저장 시 A4 표지와 본문 문서 형태로 출력
- `localStorage`에 `preInterviewInfo`, `welcomeMessage`, `questions`, `answers`, `finalLetter`, `createdAt`, `selectedLetterTone`, `selectedLetterLength`, `letterRevisionHistory` 저장

## 테스트 시나리오

1. 첫 화면에 아래처럼 입력합니다.
   - 고객 이름: 송민준
   - 대상자 이름: 박정자
   - 관계: 어머니
   - 오늘 기록의 목적: 감사
   - 가장 먼저 떠오르는 기억: 어릴 때 엄마가 안아주던 기억
   - 꼭 전하고 싶은 말: 엄마가 늘 나를 믿어줘서 고마웠다는 말
2. `입장하기`를 누릅니다.
3. 3초 뒤 환영 멘트가 나타나고 음성으로 재생되는지 확인합니다.
4. 환영 멘트가 재생되는 동안 `기억 진행자` 아바타 주변에 은은한 움직임이 보이는지 확인합니다.
5. `인터뷰 시작`을 누릅니다.
6. 질문이 음성으로 재생되는 동안 인터뷰 화면의 아바타가 다시 움직이는지 확인합니다.
7. 질문 1부터 질문 7까지 텍스트로 답변합니다.
8. 질문 화면에서 `마이크로 말하기`를 누르고 짧게 말해봅니다.
9. 버튼 문구가 `듣는 중...`으로 바뀌고, 인식된 말이 textarea 뒤에 이어 붙는지 확인합니다.
10. 다시 버튼을 눌러 음성인식이 멈추는지 확인합니다.
11. `영상 기록 시작`을 누르고 카메라/마이크 권한을 허용합니다.
12. 작은 미리보기 영상이 보이는지 확인합니다.
13. `영상 기록 중지`를 눌러 webm 영상 플레이어와 `영상 다운로드` 버튼이 나타나는지 확인합니다.
14. 각 질문에서 `다시 듣기`, `음성 끄기/켜기`를 확인합니다.
15. 7번째 답변 후 편지 톤과 길이를 선택합니다.
16. `가족에게 남기는 편지 만들기`를 눌러 최종 편지를 확인합니다.
17. `결과 페이지 열기`를 눌러 `/result/[uuid]` 페이지가 열리는지 확인합니다.
18. 결과 페이지에서 `링크 복사`, `PDF 저장`, `카카오톡 공유` 버튼을 확인합니다.
19. PDF 저장을 누른 뒤 A4 표지에 `오늘 남겨진 이야기`, 작성자, 생성일이 보이는지 확인합니다.
20. 최종 편지 화면에서도 녹화된 영상을 다시 볼 수 있는지 확인합니다.
21. 최종 편지 화면에서 `더 부드럽게`, `더 담담하게`, `짧게 줄이기` 등을 눌러 편지가 다시 생성되는지 확인합니다.
22. 새로고침 후 텍스트 기록이 유지되는지 확인합니다. 영상 파일은 서버에 업로드하지 않으므로 필요한 경우 다운로드해 보관합니다.

## 참고

현재는 실제 AI API를 연결하지 않고 브라우저 안에서 질문과 편지를 생성합니다. 음성인식, 영상녹화, 아바타 기능을 붙이기 쉽도록 화면 컴포넌트와 음성, 저장, 질문 생성 로직을 분리했습니다.
